# 핵심 기능

이 문서는 PickTrip Spring 서버가 담당하는 핵심 기능을 정의한다. 화면 구성보다 API, 도메인 모델, 데이터 신뢰성, 외부 연동 책임을 중심으로 작성한다.

## 1. 로그인 / 회원가입

일정 저장과 공유 기능은 로그인 사용자를 기준으로 제공한다. 콘텐츠 탐색은 비로그인 상태에서도 가능해야 한다.

### 서버 책임

- Kakao 소셜 로그인 연동
- Google 소셜 로그인 연동
- 외부 OAuth 사용자 식별자를 내부 사용자와 매핑
- 인증 토큰 발급, 검증, 만료 처리
- 현재 사용자 정보 조회 API 제공

### 구현 기준

- 외부 제공자 사용자 ID와 이메일은 변경 가능성을 고려해 분리 저장한다.
- 액세스 토큰, 리프레시 토큰, 세션 정책은 문서화된 만료 시간을 가진다.
- 인증이 필요한 API와 공개 API를 명확히 분리한다.

## 2. 지역 콘텐츠 큐레이션

하동, 영주, 예천의 음식, 축제, 관광지, 자연, 문화, 체험 콘텐츠를 서버에서 검색 가능한 형태로 제공한다.

### 서버 책임

- 지역별 콘텐츠 목록 조회
- 콘텐츠 상세 조회
- 키워드 검색
- 카테고리, 동행 조건, 실내·실외, 주차, 예약 필요 여부 필터
- 데이터 출처와 마지막 검증일 제공

### 콘텐츠 필드

- 장소명 또는 콘텐츠명
- 지역, 카테고리
- 한 줄 소개
- 추천 대상
- 예상 소요 시간
- 운영시간, 휴무일
- 위치, 좌표
- 주차 가능 여부
- 아이 동반 적합도
- 부모님 동반 적합도
- 실내·실외 여부
- 계절성
- 예약 필요 여부
- 축제 개최 기간
- 데이터 출처, 마지막 검증일

## 3. 여행 바구니

여행 바구니는 사용자가 선택한 콘텐츠 목록이자 AI 일정 생성의 입력값이다.

### 서버 책임

- 사용자의 여행 조건 저장
- 사용자가 담은 콘텐츠와 우선순위 저장
- 여행 바구니 조회, 추가, 제거, 우선순위 변경
- AI 일정 생성 요청 시 입력 데이터 정규화

### 우선순위

- `MUST_VISIT`: 꼭 가기
- `PREFERRED`: 가면 좋음
- `OPTIONAL`: 시간 남으면

서버는 우선순위를 단순 문자열이 아니라 명시적인 enum 값으로 관리한다.

## 4. AI 일정 생성

AI 일정 생성은 사용자의 여행 조건과 선택 콘텐츠를 기반으로 일정 후보를 생성한다.

### 서버 책임

- 일정 생성 요청 검증
- 콘텐츠, 운영시간, 축제 기간, 체류 시간, 이동 조건을 AI 입력으로 변환
- AI 응답을 서버 도메인 모델로 검증 및 정규화
- 생성된 일정과 판단 이유를 응답

### AI가 고려해야 할 요소

- 여행 날짜와 기간
- 장소별 운영시간
- 축제 개최 기간
- 식사 시간
- 장소별 예상 체류 시간
- 이동 거리와 이동 순서
- 동행 유형
- 걷기 부담
- 실내·실외 비율
- 날씨 또는 우천 대안
- 사용자가 설정한 우선순위

### 응답 기준

AI 일정 결과에는 각 장소 배치 이유를 포함한다.

- "축제 운영시간이 오전 10시부터라서 1일차 오전에 배치했습니다."
- "아이와 함께 이동하기 쉽도록 가까운 장소끼리 묶었습니다."
- "점심 시간대에 지역 대표 음식을 먹을 수 있도록 식당을 먼저 배치했습니다."

## 5. 일정 수정 및 재생성

사용자는 AI가 만든 일정을 저장 전후에 수정할 수 있어야 한다.

### 서버 책임

- 일정 저장
- 일정 상세 조회
- 장소 순서 변경
- 장소 삭제
- 대체 장소 추가
- 특정 장소 고정 여부 저장
- 전체 일정 또는 하루 단위 재생성 요청 처리

수정 이력은 MVP에서 필수는 아니지만, 마지막 수정 시각은 저장한다.

## 6. 일정 공유

저장된 일정은 외부에 공유 가능한 링크로 제공한다.

### 서버 책임

- 공유 링크 생성
- 공유 토큰으로 일정 조회
- 비공개 일정과 공유 일정 접근 정책 분리
- 공유 링크 비활성화 기능을 고려한 데이터 구조 설계

공유 토큰은 예측 불가능해야 하며 내부 일정 ID를 그대로 노출하지 않는다.

## 7. 가족 여행 편의 필터

가족 여행자의 의사결정을 돕기 위해 일반 여행 앱보다 현실적인 조건 필터를 강화한다.

### 필터 예시

- 아이와 가기 좋음
- 부모님과 가기 좋음
- 걷기 적음
- 주차 편함
- 실내 가능
- 비 오는 날 가능
- 화장실 이용 편함
- 예약 필요
- 반려동물 동반 가능 여부

필터는 콘텐츠 검색, 바구니 추천, AI 일정 생성 입력에 모두 재사용 가능한 구조로 관리한다.
