# MVP 범위

PickTrip 서버의 초기 버전은 기능을 과도하게 넓히기보다 하동, 영주, 예천 3개 지역의 핵심 여행 일정 생성 경험을 안정적으로 제공하는 데 집중한다.

## MVP 목표

- 사용자는 소셜 로그인으로 서버에 인증할 수 있다.
- 사용자는 하동, 영주, 예천 콘텐츠를 검색하고 필터링할 수 있다.
- 사용자는 콘텐츠를 여행 바구니에 담고 우선순위를 설정할 수 있다.
- 서버는 선택된 콘텐츠와 여행 조건을 바탕으로 AI 일정 생성을 요청하고 결과를 반환할 수 있다.
- 사용자는 생성된 일정을 저장하고 공유할 수 있다.

## MVP 포함 기능

- [x] Kakao 또는 Google 소셜 로그인
  - 조건: OAuth 인증 후 서버가 내부 사용자 계정을 생성하거나 조회할 수 있다.
- [x] 인증 토큰 발급과 사용자 조회
  - 조건: 로그인 후 현재 사용자 정보를 조회할 수 있다.
- [x] 하동, 영주, 예천 지역 콘텐츠 조회
  - 조건: 3개 지역 중 1개 이상을 기준으로 콘텐츠 목록을 조회할 수 있다.
- [x] 콘텐츠 검색과 필터
  - 조건: 검색어, 카테고리, 동행 조건, 실내·실외, 주차 여부 등으로 조회할 수 있다.
- [x] 콘텐츠 상세 조회
  - 조건: 운영시간, 위치, 체류 시간, 예약 필요 여부, 데이터 출처를 확인할 수 있다.
- [x] 여행 조건 저장 또는 전달
  - 조건: 지역, 날짜, 기간, 동행 조건을 일정 생성 입력값으로 사용할 수 있다.
- [x] 여행 바구니
  - 조건: 콘텐츠를 추가, 삭제, 조회할 수 있다.
- [x] 바구니 콘텐츠 우선순위 설정
  - 조건: `MUST_VISIT`, `PREFERRED`, `OPTIONAL` 중 하나로 저장할 수 있다.
- [x] AI 일정 생성
  - 조건: 2개 이상 콘텐츠 선택 시 1박 2일 기준 일정 1개 이상을 생성할 수 있다.
- [x] AI 일정 생성 이유 제공
  - 조건: 각 장소 배치 이유를 응답에 포함한다.
- [x] 일정 수정
  - 조건: 장소 순서 변경, 삭제, 대체 장소 추가, 특정 장소 고정 값을 저장할 수 있다.
- [x] 일정 저장
  - 조건: 로그인 후 저장한 일정이 앱 재시작 후에도 조회 가능하다.
- [x] 일정 공유
  - 조건: 저장된 일정을 공유 토큰 기반 링크로 조회할 수 있다.

## MVP 제외 기능

- 숙소 예약
- 결제
- 실시간 교통
- 사용자 커뮤니티
- 리뷰 시스템
- 모든 경상도 지역 확장
- 완전 자동 데이터 수집
- 관리자 CMS
- 복잡한 추천 개인화
- 다중 사용자 공동 편집

## 서버 우선순위

1. 인증과 사용자 식별
2. 콘텐츠 데이터 모델과 조회 API
3. 여행 바구니와 일정 생성 입력 모델
4. AI 일정 생성 연동과 응답 정규화
5. 일정 저장과 공유

각 단계는 독립적으로 테스트 가능해야 하며, 외부 API 연동은 실패 가능성을 전제로 예외 처리를 포함한다.

## TourAPI 수집 우선순위

- [ ] 1단계: API 클라이언트 공통 모듈 — 인증키, 공통 파라미터, JSON 파싱, 에러 처리 (선행 없음)
- [ ] 2단계: 지역 코드 seed — `/areaCode2`, `/ldongCode2` (1단계 완료 후)
- [ ] 3단계: 분류 코드 seed — `/categoryCode2`, `/lclsSystmCode2`, 내부 카테고리 매핑 (1단계 완료 후)
- [ ] 4단계: 콘텐츠 수집 배치 — `/areaBasedList2` + 상세 API 세트 (2, 3단계 완료 후)
- [ ] 5단계: 축제 수집 배치 — `/searchFestival2` (2단계 완료 후)
- [ ] 6단계: 이미지 보강 — `/detailImage2` 우선, 부족 시 `gallerySearchList1` (4단계 완료 후)
- [ ] 7단계: 동기화 배치 — `/areaBasedSyncList2`, `/gallerySyncDetailList1` (4, 6단계 완료 후)

상세 오퍼레이션 목록과 동기화 전략은 `.agents/docs/tour-api-sync.md` 참고.
