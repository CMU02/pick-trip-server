현재 브랜치의 변경된 파일을 대상으로 코드 리뷰를 수행한다.

## 검토 순서

1. `git diff main...HEAD` 로 변경 파일 목록을 확인한다.
2. 변경된 `.java` 파일 각각에 대해 아래 체크리스트를 점검한다.

## 체크리스트

### 컨벤션
- [ ] Entity에 `@Setter`, `@Data` 가 없는가
- [ ] Controller가 Entity를 직접 반환하지 않는가
- [ ] Controller에 `@Transactional` 이 없는가
- [ ] `javax.*` 대신 `jakarta.*` 를 사용하는가
- [ ] DTO가 Bean Validation 어노테이션을 포함하는가 (Request DTO)

### 품질
- [ ] 메서드가 단일 책임을 갖는가
- [ ] 예외 처리가 명시적 에러 코드를 사용하는가
- [ ] 외부 API 호출에 예외 처리가 있는가
- [ ] N+1 문제가 발생할 수 있는 연관관계 조회가 없는가

### 보안
- [ ] 서버 내부 예외 메시지를 응답에 그대로 노출하지 않는가
- [ ] 인증이 필요한 엔드포인트에 Security 설정이 되어 있는가

## 결과 보고

체크리스트 항목별로 ✅ (통과) / ⚠️ (주의) / ❌ (위반) 로 표시하고,
위반 항목은 파일명과 라인 번호를 함께 명시한다.
